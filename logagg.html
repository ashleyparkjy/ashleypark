ON-DEMAND LOG AGGREGATOR (NUTANIX INTERN HACKATHON 2021)

This was an intern hackathon project at Nutanix. With a team of 8, we built an internal log aggreagtor tool using Filebeat, Elastic Search, and Kibana. The problem to solve was that the log collection and viewing process was too long and complicated. Our proposed solution was to develop an application that can collect, forward, and prepare the data for use on demand. With our application, when a user reports an error, and an engineer wants to get logs from their cloud, the following steps happen:

  1. The customer logs into their cloud and runs the one line command that the engineer send them.
  2. Then the log collecting script runs the company's internal logbay tool to aggregate all the necessary logs into a single directory, filtered by duration, tags, etc.
  3. Then filebeat is started, forwarding the content of each log file in the directory to the external ElasticSearch endpoint, tagged with identifiers such as case id and customer id.
  4. The ElasticSearch database uses an Ingest Node Pipeline and Grok patterns to further extract useful data from the logs, such as log time and log level.
  5. Then the logs will be available on the Kibana dashboard with searching, viewing, and visualizing capabilities.

This process improved the efficiency of the log collection process and also improved the productivity of engineers on debugging customer problems by using ElasticSearch and Kibana. The biggest impact of this feature is that it cut the process by half from 6 steps to 3 steps. Originally, before even looing through piles of logs, an engineer would have to take several steps to collect and download the logs. But with this automated tool, the engineer just has to run a single internal script and all the logs will be available in Kibana after some time so that they can search and browse effectively using filters. This project won the Hackathon.
